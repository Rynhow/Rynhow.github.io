首先是框架选择，可以理解为人的骨骼，没有骨骼就支撑不起肉，是最基本的条件。

## 一、最推荐：通用均衡全能

### 1. Tailwind CSS

**定位**：原子化 CSS 框架**推荐度**：★★★★★（全能天花板）

- 优点：
  - 极度灵活，想怎么写样式就怎么写
  - 体积小、性能好，适合 Hexo 主题魔改
  - 现代、主流，几乎所有新主题都用它
- 适合：**改 Hexo、做自定义网页、做博客主题**

### 2. Bootstrap 5

**定位**：老牌全能 CSS+JS 组件框架**推荐度**：★★★★☆（最稳、最省心）

- 优点：
  - 组件超多：导航、卡片、表单、弹窗、轮播… 开箱即用
  - 兼容性极强，新手零门槛
- 适合：**快速做官网、后台页面、普通网页**

------

## 二、前端完整框架（做动态网站、WebApp）

### 1. Vue 3 + Vite

**推荐度**：★★★★★（均衡之王）

- 上手易、生态强、文档全
- 适合：个人站、工具站、中小后台、博客系统

### 2. React + Next.js

**推荐度**：★★★★★（大厂标准、全能）

- 支持 SSR/SEO 极强
- 适合：复杂网站、电商、大型项目

### 3. Astro

**推荐度**：★★★★★（静态站最强新贵）

- 比 Hexo 更强、更快、支持多框架
- 适合：**想升级博客 / 做高性能静态站**

------

## 三、轻量极简（只想快速写个页面）

- **Pure.css**：极小、无依赖
- **Bulma**：现代、无 jQuery、比 Bootstrap 轻
- **W3.CSS**：最简单，入门级

------

# 最终一句话结论

- 你是**改 Hexo 主题 / 做自定义网页** → 用 **Tailwind CSS**（最均衡全能）
- 你是**新手快速做网站** → 用 **Bootstrap 5**
- 你是**做现代前端项目** → 用 **Vue 3 + Vite**
- 你想**换比 Hexo 更强的静态站框架** → 用 **Astro**





# 一、静态博客框架（首选，免费 + 快 + 安全）

## 1. **Hexo**

**推荐度：★★★★★ 【最适合你】**

- 基于 Node.js，上手最简单

- 主题极多（Butterfly、Fluid、Next 都是顶流）

- 一键部署 GitHub Pages / Gitee Pages

- 文章用 Markdown，纯文本好管理

  

  适合

  ：只想快速搭好、长期稳定用的个人博客

## 2. **Hugo**

**推荐度：★★★★☆**

- Go 编写，**生成速度极快**（千篇文章秒生成）

- 比 Hexo 更轻、更稳

- 缺点：配置语法比 Hexo 稍复杂

  

  适合

  ：文章很多、追求极致速度的人

## 3. **Astro**

**推荐度：★★★★☆**

- 现代新一代静态框架

- 支持 Vue/React 组件，性能极强

- 做博客、作品集、文档站全能

  

  适合

  ：想玩现代前端、博客要很炫酷的人

## 4. **VitePress**

**推荐度：★★★★**

- Vue 官方出品，极轻极快

- 偏「文档 + 博客」，界面极简清爽

  

  适合

  ：技术笔记、文档型博客

# Hexo 搭建个人博客（完整速查版）

**核心结论**：按以下步骤可在**10 分钟内**完成 Hexo + GitHub Pages 博客搭建，默认使用 **Butterfly** 简约主题，全程可直接复制命令执行。

------

### 一、环境准备（必装）

表格







| 工具    | 版本要求           | 安装命令 / 方式 | 验证命令      |
| ------- | ------------------ | --------------- | ------------- |
| Node.js | ≥18.0（建议 20.x） | 官网下载安装    | node -v       |
| Git     | 最新稳定版         | 官网下载安装    | git --version |

- 下载地址：
  - Node.js：https://nodejs.org/（长期支持版）
  - Git：https://git-scm.com/

------

### 二、安装 Hexo

bash



运行









```bash
# 全局安装 Hexo CLI
npm install -g hexo-cli

# 验证安装（输出版本号即成功）
hexo -v
```

------

### 三、初始化博客

bash



运行









```bash
# 创建博客目录并进入（名称自定义，如 hexo-blog）
mkdir hexo-blog && cd hexo-blog

# 初始化 Hexo 项目
hexo init

# 安装项目依赖
npm install
```

------

### 四、本地预览（测试）

bash



运行









```bash
# 生成静态文件
hexo g  # 等价 hexo generate

# 启动本地服务器
hexo s  # 等价 hexo server
```

- 访问：http://localhost:4000，看到默认博客页面即成功
- 停止服务：Ctrl + C

------

### 五、Butterfly 主题配置（推荐）

#### 1. 下载主题

bash



运行









```bash
# 克隆 Butterfly 主题到 themes 目录
git clone -b master https://github.com/jerryc127/hexo-theme-butterfly.git themes/butterfly
```

#### 2. 安装依赖（必装）

bash



运行









```bash
# 安装渲染器（否则主题报错）
npm install hexo-renderer-pug hexo-renderer-stylus --save
```

#### 3. 启用主题

编辑博客根目录 **_config.yml**（站点配置文件）：

yaml











```yaml
# 找到 theme 字段，修改为 butterfly
theme: butterfly
```

#### 4. 主题个性化配置

复制主题配置文件到根目录（避免更新覆盖）：

bash



运行









```bash
cp themes/butterfly/_config.yml _config.butterfly.yml
```

编辑 **_config.butterfly.yml**，可修改：

- 网站标题、描述、头像
- 导航栏菜单
- 社交链接（GitHub、知乎等）
- 代码高亮样式

------

### 六、部署到 GitHub Pages（免费托管）

#### 1. 创建 GitHub 仓库

1. 登录 GitHub，新建仓库
2. 仓库名必须为：**你的用户名.github.io**（例：zhangsan.github.io）
3. 选择「公开」，点击创建

#### 2. 配置部署信息

编辑博客根目录 **_config.yml**，添加 / 修改 deploy 字段：

yaml











```yaml
# 部署配置
deploy:
  type: git
  repository: https://github.com/你的用户名/你的用户名.github.io.git  # 或 SSH 地址
  branch: main  # 旧仓库可能是 master
```

#### 3. 安装部署插件

bash



运行









```bash
npm install hexo-deployer-git --save
```

#### 4. 执行部署

bash



运行









```bash
# 生成文件并部署（二合一命令）
hexo d -g  # 等价 hexo generate && hexo deploy
```

#### 5. 访问博客

浏览器输入：https:// 你的用户名.github.io，即可访问

### 九、常见问题排查

1. **主题不生效**：检查主题文件夹名称是否为 **butterfly**，且站点配置文件 `theme: butterfly` 无拼写错误
2. **部署后 404**：确认仓库名格式正确，分支为 main，且 GitHub Pages 已启用（仓库 Settings → Pages → 选择 main 分支）
3. **依赖报错**：执行 `npm install` 重新安装依赖，或删除 `node_modules` 和 `package-lock.json` 后重试
4. **中文乱码**：确保文件编码为 UTF-8，站点配置文件添加 `language: zh-CN`

------

### 十、进阶优化（可选）

1. **配置域名**：在仓库 Settings → Pages 中绑定自定义域名，并在域名解析后台添加 CNAME 记录
2. **开启评论**：Butterfly 支持 Waline、Valine 等评论系统，在 `_config.butterfly.yml` 中配置
3. **SEO 优化**：安装 `hexo-generator-sitemap` 和 `hexo-generator-baidu-sitemap` 插件，生成站点地图提交给搜索引擎
