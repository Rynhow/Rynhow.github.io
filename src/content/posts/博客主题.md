# Astro主题

## AstroPaper

优点：

- 它是Astro官方推荐的主题，性能满分，极致简约，特别是性能，流畅至极
- 支持高度自定义，响应式布局，不用去刻意去写移动端的适配

不足：

- 它对我来说太简约了
- 不支持一键汉化，如果想汉化那只能去改组件
- 没有toc组件，对我来说这个比较重要

官方链接：https://github.com/satnaing/astro-paper



## Firefly

这是一个基于fuwari模板的二改主题，也是现在这个网站用的主题

优点：

- 高度自定义，美观，页面过度平滑
- 自带Toc，音乐播放器等组件
- 支持改i18n一键切换语言

不足：

暂时还没发现

官方链接：https://github.com/CuteLeaf/Firefly



## Mizuki

这也是一个基于fuwari模板的二改主题，但和Firefly侧重点不同

优点：

- 自带了多种美观的字体
- 组件很多，但我用不到那么多的组件，例如日记，相册等
- 其他的和Firefly差不多

不足：

- 个人感觉有点“重”，因为组件多，也会比Firefly加载慢

喜欢折腾组件的可以试试

官方链接：https://github.com/LyraVoid/Mizuki

# hexo主题

## Butterfly

Hexo 生态中**最主流、最活跃**的现代化主题，GitHub 星标 8000+，社区支持完善，功能全面且高度可定制。

优点：

- 功能极其丰富，支持暗色模式、多语言、评论系统、站内搜索等几乎所有博客所需功能
- 主题文档完善，中文社区活跃，遇到问题容易找到解决方案
- 设计优雅，视觉层次清晰，注重阅读体验，适配各种屏幕尺寸
- 支持插件化扩展，可通过 npm 安装额外功能模块，满足个性化需求

不足：

- 功能太多导致配置复杂，新手需要一定学习成本
- 主题体积较大，默认配置下加载速度一般，需要手动优化
- 部分高级定制需要修改 Stylus/CSS 源码，对非前端用户不够友好

适合希望**一站式解决所有博客需求**、愿意投入时间学习配置、需要长期维护的博客作者，尤其适合内容丰富的综合性博客。

官方链接：https://github.com/jerryc127/hexo-theme-butterfly



## Next

Hexo 生态中的**经典主题**，经过多年迭代，稳定性与兼容性极佳，适合注重传统博客体验的用户。

优点：

- 稳定性强，与 Hexo 各版本兼容性好，几乎不会出现兼容性问题
- 配置简洁明了，新手容易上手，文档全面且易懂
- 支持多种布局（单栏 / 双栏）、多种评论系统，满足基础博客需求
- 对 SEO 友好，默认配置下即可获得良好的搜索引擎排名

不足：

- 设计风格相对传统，缺乏现代感，视觉冲击力较弱
- 功能更新较慢，部分新特性（如暗色模式）需要手动配置
- 主题定制化程度不如 Butterfly 等现代主题

适合**追求稳定可靠**、不喜欢频繁折腾、内容以技术文章为主的作者，尤其适合长期维护的技术博客。

官方链接：https://github.com/next-theme/hexo-theme-next
